# ホーム検索UIのタブ化(サウナイキタイ風) 実装指示書 (2026-07-07)

実装エージェント向けの指示書。DESIGN.md・READMEの規約に従うこと。
pnpmがクラッシュする環境のため、コマンドはすべて `npm run ...` を使う。

## ゴール

トップページのヒーロー検索を、サウナイキタイのトップと同じ「3タブ切り替え型」にする。

- タブ: **キーワードからさがす / 都道府県からさがす / 特徴からさがす**
- タブを切り替えると、タブバー直下のパネルの中身だけが切り替わる(ページ遷移なし)
- 都道府県・特徴はリンク集であり、クリックで `/spots?...` に遷移する(フォーム送信ではない)

## 参考: サウナイキタイのデザイン仕様(観察結果)

- ヒーロー中央、ロゴの下に横幅いっぱい(max-width制約あり)のタブバー。3タブが等幅で並ぶ
- タブバーは背景色の上に白の枠線ボックス。**アクティブタブだけ塗りつぶし反転**(白背景+背景色の文字)、非アクティブは透過背景+白文字
- アクティブタブの直下に小さな**下向き三角のノッチ**が付き、どのタブが開いているか視覚的につながる
- パネルの中身:
  - キーワード: 大きめの入力欄 + 右端に検索ボタンが一体化した1行フォーム
  - 都道府県: 「北海道・東北 / 関東 / 北陸・甲信越 / 東海 / 関西 / 中国・四国 / 九州・沖縄」の地方見出しごとに、都道府県名のテキストリンクが横に流れる。件数表示はなし
  - 特徴: 特徴名のリンク(チップ)がカテゴリごとに並ぶ。クリックでその条件の検索結果へ

## どこランへの翻案(このリポジトリでの仕様)

### 1. コンポーネント分割

現状 [src/app/page.tsx](../src/app/page.tsx) はサーバーコンポーネント1ファイルで、17〜21行目に `<form action="/spots">` のヒーロー検索がある。タブ切り替えはクライアント状態が要るので、次の構成にする。

- 新規 `src/components/hero-search.tsx` — `"use client"`。タブ状態(`useState<"keyword" | "pref" | "feature">`)を持つ
- `page.tsx` はサーバーコンポーネントのまま、`getTags()` と `getPrefectureCounts()` の結果を props で `HeroSearch` に渡す(どちらも既存: [src/db/data.ts](../src/db/data.ts))
- 既存のヒーローの見出し・コピー(「次はどこでランする？」等)はそのまま残し、フォーム部分だけを `HeroSearch` に置き換える

### 2. タブバーのデザイン

ヒーロー背景は `bg-brand`(黄色 #ffd900)なので、サウナイキタイの「白枠+反転」を ink 基調に置き換える。

- タブバー: `grid grid-cols-3`、`rounded-xl border-2 border-ink overflow-hidden`(角丸の外枠で3分割)
- 非アクティブタブ: 透過背景 + `text-ink font-bold`
- アクティブタブ: `bg-ink text-white`(反転)
- アクティブタブ直下に下向き三角のノッチ(CSSのborderで作る小さな▼、色は `border-t-ink`)
- タブは `<button type="button">`。`role="tablist"` / `role="tab"` / `aria-selected` / `aria-controls` を付け、パネル側に `role="tabpanel"` を付ける
- モバイルでも3等分のまま。ラベルは「キーワード / 都道府県 / 特徴」に短縮してもよい(sm以上でフル表記)

### 3. パネル共通

- タブバーの直下に白いカード(`rounded-2xl border border-line bg-paper p-4 sm:p-5 shadow-lg`)。現行フォームの見た目を踏襲
- 3パネルとも常にDOMに描画し、非アクティブは `hidden` にする(リンクがSSR HTMLに含まれ、SEO内部リンクとして効く)
- 現行の `sm:absolute sm:-bottom-12` の浮かせレイアウトは、パネルの高さが可変になるため廃止してよい。ヒーロー内の通常フローに置き、下部コンテンツとの余白を調整する

### 4. キーワードパネル

- 現行フォームを簡素化: `<form action="/spots">` + `q` 入力 + 「さがす」ボタンの1行(虫めがねアイコンは現行踏襲)
- **都道府県 `<select>` は削除**(都道府県タブに役割を移す)
- プレースホルダは現行どおり「スポット名・市区町村」

### 5. 都道府県パネル

- [src/lib/prefectures.ts](../src/lib/prefectures.ts) の `regionGroups` を使い、地方名を小見出しにして都道府県リンクを横に流す(サウナイキタイと同じ構造)
- リンク先は現行どおり `/spots?pref=<都道府県名>`(URLエンコード)。`/pref/xxx` の新ルートは**作らない**
- どこランは掲載ゼロの県があるため、サウナイキタイと違い次の扱いにする:
  - 掲載ありの県: リンク + `(件数)` を表示(トップ下部「エリアからさがす」の現行仕様を踏襲)
  - 掲載ゼロの県: 非表示(グレー表示ではなく出さない)。地方内が全部ゼロならその地方ごと非表示
- 件数は `getPrefectureCounts()` の結果を props で受け取る

### 6. 特徴パネル

`getTags()` の結果をタグの `category` でグルーピングして表示する。カテゴリ定義は [src/db/schema.ts](../src/db/schema.ts) の enum(`terrain` / `environment` / `scenery`)。

- カテゴリ見出し(日本語ラベルを定数で定義):
  - `terrain` → 「コース・地形」
  - `environment` → 「走りやすさ」
  - `scenery` → 「景色・ロケーション」
- 各タグは既存の [TagChip](../src/components/tag-chip.tsx) を流用(リンク先 `/spots?tags=<slug>`)。見た目が合わなければ同等のチップリンクを `hero-search.tsx` 内に書いてよい
- さらに、タグ以外の既存フィルタもチップとして同列に並べる(4つ目のグループ「設備・タイプ」):
  - トイレあり → `/spots?toilet=1`
  - ロッカーあり → `/spots?locker=1`
  - 銭湯・サウナが近い → `/spots?sento=1`
  - 周回コース → `/spots?type=loop`
  - トラック → `/spots?type=track`
  - これらのクエリは [src/app/spots/page.tsx](../src/app/spots/page.tsx) と [src/db/data.ts](../src/db/data.ts) の `searchConditions` が既に解釈できるものだけを使う。新しい検索パラメータは追加しない

### 7. 既存セクションの整理

- トップ下部の「タグからさがす」セクション([page.tsx](../src/app/page.tsx) 24行目)は特徴タブと完全に重複するので**削除**
- 「エリアからさがす」セクション(27行目)は都道府県タブと重複するが、ページ下部のSEO・回遊導線として**残す**(変更不要)
- 「人気スポット」「新着スポット」は変更不要

## やらないこと

- `/pref/<slug>` のような都道府県静的ページの新設
- 検索API・DBスキーマ・タグマスタの変更
- `/spots` 側のフィルタUI([search-filters.tsx](../src/components/search-filters.tsx))の改修(別タスク)

## 受け入れ条件

1. `npm run build` が通る。hydrationエラーがコンソールに出ない
2. 3タブが切り替わり、リロード直後は「キーワードからさがす」がアクティブ
3. キーワード検索: `q` 入力 → 送信で `/spots?q=...` に遷移し結果が出る
4. 都道府県タブ: 掲載ありの県だけが地方ごとに件数つきで並び、クリックで `/spots?pref=...` に遷移して正しく絞り込まれる
5. 特徴タブ: タグ17件がカテゴリ3分類+「設備・タイプ」で並び、クリックで絞り込み結果に遷移する
6. モバイル(375px)でタブ・パネルが崩れない。タブはキーボード(Tab + Enter)で操作できる
7. 非アクティブパネルのリンクもSSRのHTMLソースに含まれている(`curl` で確認)
