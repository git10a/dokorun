# スポット一覧ページのサウナイキタイ風リデザイン 実装指示書 (2026-07-07)

実装エージェント向けの指示書。DESIGN.md・READMEの規約に従うこと。
pnpmがクラッシュする環境のため、コマンドはすべて `npm run ...` を使う。
他エージェントが並行で別ファイルを編集している可能性があるため、この指示書に書かれたファイル以外はむやみに変更しないこと。

## ゴール

`/spots`(スポット一覧・検索結果ページ)を、サウナイキタイの検索ページ https://sauna-ikitai.com/search と同じ情報設計・雰囲気にリデザインする。

一言でいうと: **「グリッドに並ぶ小さなカード」から「縦に流れる横長の情報リッチなカード」へ**。写真は目を引く入口、右側にスペック数値(距離・高低差・信号)が数字ドンで並び、1枚見れば走りに行くか判断できるカードにする。

## 参考: サウナイキタイ検索ページのデザイン仕様(観察結果)

- ページ上部に検索フォーム(地域 / 条件 / キーワードが横並び + 「絞り込み条件をすべて表示」で詳細パネル展開)
- その下に **「検索結果 15992件」の件数表示** と **並び替えドロップダウン** が同じ行に並ぶ
- 結果は **1カラムの縦リスト**。カードは横長で:
  - 左に施設写真(カード高さいっぱい、幅固定)
  - 右に情報群: 施設名(リンク・大きめ太字) → 施設タイプ+地域(小さめグレー)
  - その下に **スペック数値ブロック**: サウナ温度「105℃」水風呂温度「15℃」が、アイコン+大きい数字で横に並ぶ。数字が主役で単位は小さい
  - 下部に料金「900円〜」・営業時間・「イキタイ 37079」「サ活 34515」のカウント
- カードは白背景・薄い枠線・角丸。ページ背景は白〜薄グレーでカードが浮く
- ページネーションは数字リンクの横並び

## どこランへの翻案(このリポジトリでの仕様)

サウナイキタイの語彙をどこランに置き換える:

| サウナイキタイ | どこラン |
|---|---|
| サウナ温度 105℃ | 距離 **5.0**km |
| 水風呂温度 15℃ | 高低差 **12**m |
| 外気浴・ロウリュ | 信号 **0** |
| 施設タイプ+地域 | 都道府県+市区町村 |
| イキタイ/サ活数 | (フェーズ2のためまだ無い。**表示しない**) |
| 料金・営業時間 | コース形状・路面(周回 / 舗装路 など) |

カラーはDESIGN.md §9 のトークンをそのまま使う(黄 `brand` × 黒 `ink` × 白 `paper`)。サウナイキタイの青は真似せず、強調は `brand` / `brand-dark` で行う。

### 1. データ層: SpotSummary に高低差と信号数を追加

カードのスペック表示に `elevationGainM` と `signalsCount` が要る。どちらも `courses` テーブルに既にあるが `SpotSummary` に入っていない。

- [src/lib/types.ts](../src/lib/types.ts): `SpotSummary` に `elevationGainM: number | null` と `signalsCount: number | null` を追加
- [src/db/data.ts](../src/db/data.ts): `summarySelection` に `elevationGainM: courses.elevationGainM` と `signalsCount: courses.signalsCount` を追加。`decorate` 関数のジェネリック制約にも同フィールドを追加
- `MapSpot` は変更しない

### 2. SpotCard を横長カードに全面リデザイン

対象: [src/components/spot-card.tsx](../src/components/spot-card.tsx)

現状は「上に写真・下にテキスト」の縦カード。これを横長カードに作り替える。

```
+-------------+--------------------------------------------+
|             | 東京都 千代田区                              |
|   写真      | 皇居                                        |
| (左端固定幅) |  [5.0km] [高低差12m] [信号0]  ← 数字が主役    |
|             | 周回・舗装路                                 |
|             | #フラット #夜も明るい  🚻🔒♨ (設備アイコン)   |
+-------------+--------------------------------------------+
```

- ルート要素は現状どおり `<Link href={`/spots/${spot.slug}`}>` 1枚まるごとリンク。`rounded-xl border border-line bg-paper hover:-translate-y-0.5 hover:shadow-md` の系統も踏襲
- レイアウト: `flex`。写真は `w-32 sm:w-56 shrink-0 self-stretch`(モバイルでも横長を維持する。サウナイキタイもモバイルで写真左のまま)。`overflow-hidden` で角丸からはみ出さないこと
- 写真が無い場合は現行どおり `SpotVisual` をフォールバック表示(削除しない)
- 右側の情報ブロック(上から順に):
  1. `text-xs text-sub` で「{prefecture} {city}」
  2. `text-lg sm:text-xl font-bold` でスポット名
  3. **スペック数値ブロック**(このカードの主役。サウナイキタイの温度表示に相当):
     - 「距離」「高低差」「信号」の3項目を横並び(`flex gap-4 sm:gap-6`)
     - 各項目は「小さいラベル(text-[11px] text-sub) + 大きい数値(text-xl sm:text-2xl font-bold) + 小さい単位(text-xs)」の構成。距離の数値のみ `text-brand-dark` で色を付ける(現行カード踏襲)
     - 距離: `(distanceM / 1000).toFixed(distanceM % 1000 ? 1 : 0)` + "km"(現行ロジック流用)
     - 高低差: `elevationGainM` + "m"。null なら数値部分を "−" にする(項目ごと消さない。3項目並んでいることが視覚的リズムを作る)
     - 信号: `signalsCount`。0 のときも "0" と表示する(信号ゼロはどこランの最大の売り)。null なら "−"
  4. `text-sm` で「{courseTypeLabels[courseType]}・{surfaceLabels[surface]}」
  5. タグチップ(現行の `rounded-full bg-cream px-2.5 py-1 text-xs`、`slice(0, 3)`)と `FacilityIcons`(compact)を1行にまとめる(`flex flex-wrap items-center gap-1.5`)。モバイルで窮屈ならタグは2個に減らしてよい
- lucide-reactのアイコンをスペック項目に添えるのは任意。付けるなら距離=`Route`、高低差=`Mountain` 程度にとどめ、数字より目立たせない

`SpotCard` は `/spots` 以外(トップページの人気スポット等)でも使われている可能性がある。`grep` で全使用箇所を確認し、トップページなど縦カードのままが適切な場所があれば、そこは新しい横長カードで崩れないか確認して調整すること(propsでバリアントを分けるより、全箇所横長に統一するほうがシンプルで望ましい。ただしグリッド側のクラス調整は必要 → §3)。

### 3. 一覧ページのレイアウト変更

対象: [src/app/spots/page.tsx](../src/app/spots/page.tsx)

- カードリストを `grid gap-5 sm:grid-cols-2` → **1カラムの縦リスト**(`space-y-4` 程度)に変更。これがサウナイキタイの見た目の核
- 右カラムの地図(`SpotsMapShell`)は現状維持。`md:grid-cols-[minmax(0,3fr)_minmax(340px,2fr)]` の2カラム構成も維持
- 件数表示の行をサウナイキタイ風に:
  - 左: 「**{total}**件のスポット」(現行踏襲。数字は `text-2xl`)
  - 右: **並び替えドロップダウン**を新設(§4)
- 0件時の表示・ページネーションは現状のロジックを維持(見た目の微調整は自由)

### 4. 並び替えUIを件数行に移動

サウナイキタイは並び替えが結果件数の右にあり、変更すると即反映される。同じ挙動にする。

- 新規 `src/components/sort-select.tsx` — `"use client"`。`useRouter` + `useSearchParams` を使い、`<select>` の change で `sort` パラメータを書き換えて `router.push`。このとき **`page` パラメータは削除**する(並び替え変更時は1ページ目に戻る)
- 選択肢は現行と同じ3つ: 新着順(`new`・デフォルト) / 距離が短い順(`distance_asc`) / 距離が長い順(`distance_desc`)
- 見た目: `h-10 rounded-lg border border-line bg-paper px-3 text-sm`
- [src/components/search-filters.tsx](../src/components/search-filters.tsx) の詳細パネル内にある「並び順」`<select>` は**削除**する(役割が移るため)。削除後、`sm:grid-cols-3` のグリッドが2項目になるので `sm:grid-cols-2` に直す。フォーム送信時に `sort` が失われないよう、`params.sort` があれば `<input type="hidden" name="sort" ...>` をフォームに入れること

### 5. 検索フォームの微調整(スコープ最小)

`SearchFilters` の構造は既にサウナイキタイ型(横並び+展開パネル)なので大きく触らない。以下のみ:

- 「条件を絞り込む」の `<summary>` をサウナイキタイの「絞り込み条件をすべて表示」風に、開閉が分かるシェブロン付きのボタン風スタイルにする(`flex items-center gap-1 font-bold` + lucide `ChevronDown`。CSSだけで `details[open]` 時に回転させる)
- それ以外(タグ・距離・設備・hiddenの`tags`同期スクリプト)は変更しない

## 受け入れ条件

1. `/spots` でカードが1カラム縦リストの横長カードで表示され、各カードに 距離・高低差・信号 の3スペックが数字主体で並ぶ
2. 高低差・信号が null のスポットでも "−" 表示でレイアウトが崩れない。信号0件は "0" と表示される
3. 並び替えドロップダウンが件数行の右にあり、変更すると即座にURLが変わって再検索される(`page` はリセット)
4. 絞り込みフォームから「並び順」が消えており、絞り込み実行後も選択中の並び順が維持される
5. 写真なしスポットは従来どおり `SpotVisual` の生成ビジュアルが写真枠に出る
6. モバイル幅(375px)でカードが横長のまま崩れない(写真 `w-32`)
7. `SpotCard` を使う全ページ(トップの人気スポット等)で表示が崩れていない
8. `npm run lint` と `npm run build` が通る。`npm run test` が既存テストを壊していない
